# Project Context & Design Standards

## Design Philosophy (设计理念)
- **Style:** 极简现代 (Minimalist Modern)
- **Reference:** Apple, Braun, Linear, Stripe (功能主义风格)
- **Grid:** 严格遵循网格系统，强调对齐与留白。
- **Typography:** 优先使用系统字体或高可读性无衬线字体。

## Stakeholders (干系人)
- **Boss:** 产品经理 (PM) - 确认商业目标与需求优先级。
- **Partner:** 前端开发 (FE) - 确认实现可行性与性能影响。
- **Verifier:** 测试工程师 (QA) - 依据设计稿验收视觉还原度。

## Expectations (期望)
- **PM:** 设计需提升转化率与操作效率，不偏离 PRD。
- **FE:** 设计稿标注清晰，资源可复用，便于组件化。
- **QA:** 提供视觉还原度检查标准（如像素偏差容忍度）。
